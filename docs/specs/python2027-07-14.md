# Python trade DataFrame API

Status: agreed  
Agreement date: 2026-07-14

## Goal

Provide a small, notebook-friendly Python API that accepts an already-loaded
pandas DataFrame of trades and returns the feature-vector state immediately
after every trade. File loading and persistence stay outside `fiml`.

The canonical input shape matches `notebooks/trades.csv`:

```text
symbol,ts,price,volume
BTCUSDT,1783950118926,1,3
```

## Public API

```python
extractor = fiml.FeatureExtractor(feature_set, output_dtype="float32")

features = extractor.compute_features(
    trades,
    symbol="symbol",
    time="ts",
    price="price",
    volume="volume",
)
```

`compute_features` is trade-only. The old `source` and bars-only `close`
arguments are removed. Low-level `update` and `transform` continue supporting
all event kinds.

All four column arguments are optional and default to the names shown above.
They must name distinct, existing columns. Custom symbol and timestamp names
are preserved in the output.

## Input contract

- The input must be a pandas DataFrame with unique column labels.
- `symbol` values are non-empty strings. They are never coerced with `str`.
- `time` values are signed-`int64` Unix timestamps in milliseconds. Datetimes,
  floats, and strings are rejected; no implicit timestamp conversion occurs.
- `price` and `volume` columns may contain integers or floats. Values are
  converted to `float64` internally and must be finite and strictly positive.
- Multiple symbols are supported.
- Timestamps must be globally nondecreasing in DataFrame order. Equal
  timestamps are allowed and preserve input order.
- Global ordering spans repeated `compute_features` calls and calls to
  low-level `update` and `transform` on the same extractor.
- The complete input is validated before the first event is dispatched. A
  failed call leaves indicator state and the timestamp watermark unchanged.
- Validation errors identify the zero-based row position, pandas index label,
  and offending column where applicable.

An empty DataFrame succeeds, returns the expected empty output shape and
dtypes, and does not lock or mutate the extractor.

## Output contract

The result is a pandas DataFrame with the input index preserved. Its columns
are:

1. the copied symbol column;
2. the copied timestamp column;
3. numeric feature columns in `feature_names()` order.

Each row is the complete feature-vector snapshot immediately after the
corresponding trade. In a multi-symbol stream, features for symbols not touched
by that row retain their latest state. Feature cells remain `NaN` until their
indicator warms up.

Moving averages declare the precise field they consume with `source="price"`,
`"volume"`, `"trade_price"`, or `"trade_volume"`. The source determines both
the dispatch route and payload field and is stored explicitly in `FeatureSet`
JSON. Because `compute_features` emits trades, use `trade_price` or
`trade_volume`; the builder default remains `price`.

Metadata is not part of the numeric feature vector: `feature_names()` and
`n_features()` continue to describe feature columns only. Feature names must
be unique and must not collide with the selected metadata column names.

## Output dtype

The extractor always calculates and stores indicator state as `float64`.
`output_dtype` controls only arrays and feature columns returned by `values`,
`transform`, and `compute_features`.

Accepted constructor and setter values are `"float32"`, `"float64"`,
`numpy.float32`, and `numpy.float64`. The default is `"float64"`; the property
getter always returns the canonical string.

`output_dtype` may be changed until the first event is successfully dispatched.
After that, assignment raises `ValueError`. Empty and failed calls do not lock
it. Both construction paths accept it:

```python
fiml.FeatureExtractor(feature_set, output_dtype="float32")
fiml.FeatureExtractor.from_json(json, output_dtype=numpy.float32)
```

## Packaging and verification

- The Python package requires Python 3.12 or newer and uses `abi3-py312`.
- DataFrame support is installed through the `pandas` extra with
  `pandas>=2.0`.
- Python regression tests cover validation, atomicity, ordering, dtype,
  metadata, multi-symbol snapshots, and warmup.
- `notebooks/test.ipynb` is an executable end-to-end example over
  `notebooks/trades.csv`.
- Pull-request CI builds and installs the wheel on Python 3.12, runs the Python
  tests, and executes the notebook without committing generated output.
